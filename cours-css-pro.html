9. CSS : comprendre la mise en forme

CSS signifie Cascading Style Sheets. Il sert à styliser le HTML.

Exemple simple

HTML :

<h1 class="hero-title">Votre site web livré rapidement</h1>

CSS :

.hero-title {
  color: #7c3aed;
  font-size: 48px;
  line-height: 1.1;
}

Où écrire le CSS ?

1. CSS dans le HTML

<h1 style="color: red;">Titre</h1>

À éviter pour un vrai projet.

2. CSS dans une balise style

<style>
  h1 {
    color: red;
  }
</style>

Possible pour tester, mais pas idéal.

3. CSS dans un fichier externe

<link rel="stylesheet" href="assets/css/style.css">

C’est la meilleure méthode.

10. Les sélecteurs CSS

Un bon CSS commence par de bons sélecteurs. L’objectif n’est pas seulement de “faire marcher” le style, mais de pouvoir le modifier plus tard sans casser toute la page.

Sélecteur de balise

h1 {
  color: #111827;
}

S’applique à tous les h1.

Sélecteur de classe

.btn {
  padding: 12px 20px;
  border-radius: 999px;
}

HTML :

<a class="btn" href="#contact">Demander un devis</a>

Sélecteur d’ID

#contact {
  padding: 80px 0;
}

HTML :

<section id="contact"></section>

À utiliser avec modération. Pour le style, préfère les classes.

Sélecteur descendant

.hero p {
  color: #4b5563;
}

Cible les paragraphes à l’intérieur de .hero.

Sélecteur direct

.card > h3 {
  margin-bottom: 12px;
}

Cible uniquement le h3 directement enfant de .card.

Pseudo-classes

.btn:hover {
  transform: translateY(-2px);
}
input:focus {
  outline: 2px solid #7c3aed;
}

Spécificité : pourquoi une règle gagne

Quand deux règles ciblent le même élément, le navigateur choisit la plus spécifique. Un ID gagne souvent contre une classe, et une classe gagne souvent contre une balise.

p {
  color: gray;
}

.intro {
  color: blue; /* gagne contre p */
}

#lead {
  color: red; /* gagne contre .intro */
}

HTML :

<p id="lead" class="intro">Ce texte sera rouge.</p>
Dans un projet pro, évite de styliser avec des ID. Les classes sont plus souples et plus faciles à réutiliser.

Pseudo-éléments

.hero-title::after {
  content: "";
  display: block;
  width: 80px;
  height: 4px;
  background: #7c3aed;
  margin-top: 16px;
}

11. Couleurs, typographie, espacements

Couleurs

:root {
  --color-primary: #7c3aed;
  --color-secondary: #2563eb;
  --color-bg: #f8fafc;
  --color-text: #111827;
  --color-muted: #6b7280;
}

Utilisation :

body {
  background: var(--color-bg);
  color: var(--color-text);
}

Typographie

body {
  font-family: system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
  font-size: 16px;
  line-height: 1.6;
}

Titres

h1,
h2,
h3 {
  line-height: 1.1;
  letter-spacing: -0.03em;
}

Espacements

.section {
  padding: 96px 24px;
}

Largeur de contenu

.container {
  width: min(1120px, calc(100% - 48px));
  margin-inline: auto;
}

Cette ligne veut dire :

  • largeur maximum : 1120 px,
  • sinon largeur de l’écran moins 48 px,
  • centré automatiquement.

Exemple concret : créer une section propre

Voici une base réutilisable pour toutes les grandes sections d’un site.

.section {
  padding: 80px 24px;
}

.container {
  max-width: 1120px;
  margin-inline: auto;
}

.section-title {
  max-width: 700px;
  margin-bottom: 32px;
}

.section-title p {
  color: var(--color-muted);
}

Démo section

Services web

Un titre court, un paragraphe limité en largeur et assez d’espace autour du bloc rendent la lecture plus professionnelle.

13. Display : block, inline, inline-block

Block

Un élément block prend toute la largeur disponible.

Exemples :

  • div
  • section
  • p
  • h1
section {
  display: block;
}

Inline

Un élément inline prend seulement la place de son contenu.

Exemples :

  • span
  • a
  • strong

Inline-block

Prend la taille du contenu, mais accepte largeur, hauteur, padding.

.tag {
  display: inline-block;
  padding: 6px 12px;
}

Exemple concret : lien transformé en bouton

Un lien est inline par défaut. Si tu ajoutes seulement du padding, le résultat peut être difficile à contrôler. Avec inline-flex ou inline-block, le lien devient un vrai bouton visuel.

.cta-link {
  display: inline-flex;
  align-items: center;
  justify-content: center;
  padding: 12px 18px;
  border-radius: 999px;
  background: #111827;
  color: white;
}

Display en pratique

inlineinline-blockinline-flex

14. Flexbox

Flexbox sert à aligner des éléments sur un axe.

Exemple : aligner logo et menu

HTML :

<header class="site-header">
  <a class="logo" href="#">ChronoSite</a>

  <nav class="nav">
    <a href="#services">Services</a>
    <a href="#pricing">Tarifs</a>
    <a href="#contact">Contact</a>
  </nav>
</header>

CSS :

.site-header {
  display: flex;
  align-items: center;
  justify-content: space-between;
  padding: 20px 24px;
}

Explication ligne par ligne

  • .site-header cible l’en-tête complet du site.
  • display: flex place le logo et la navigation sur la même ligne.
  • align-items: center centre verticalement le logo et les liens.
  • justify-content: space-between met le logo à gauche et le menu à droite.
  • padding: 20px 24px ajoute de l’espace intérieur dans le header : 20 px en haut/bas, 24 px à gauche/droite.

Propriétés principales Flexbox

PropriétéRôle
display: flexActive Flexbox
flex-directionDirection des éléments
justify-contentAlignement horizontal
align-itemsAlignement vertical
gapEspace entre les éléments
flex-wrapPermet le retour à la ligne
flexContrôle la largeur flexible d’un enfant

Exemples

.cards {
  display: flex;
  gap: 24px;
  flex-wrap: wrap;
}
.card {
  flex: 1 1 280px;
}

Cela signifie :

  • la carte peut grandir,
  • elle peut rétrécir,
  • sa base idéale est 280 px.

Situation concrète

Tu as 3 cartes de services.

<div class="service-grid">
  <article class="service-card">Site vitrine</article>
  <article class="service-card">Landing page</article>
  <article class="service-card">Maintenance</article>
</div>
.service-grid {
  display: flex;
  gap: 24px;
  flex-wrap: wrap;
}

.service-card {
  flex: 1 1 260px;
}

Résultat :

  • sur grand écran : 3 cartes côte à côte,
  • sur petit écran : les cartes passent à la ligne.

Démo Flexbox

Site vitrine
Landing page
Maintenance

15. CSS Grid

CSS Grid sert à créer des grilles en lignes et colonnes.

Exemple simple

.services {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 24px;
}

Explication ligne par ligne

  • .services cible la section qui contient les cartes de services.
  • display: grid active CSS Grid sur ce conteneur.
  • grid-template-columns: repeat(3, 1fr) crée 3 colonnes identiques.
  • repeat(3, ...) évite d’écrire trois fois la même valeur.
  • 1fr signifie “une fraction de l’espace disponible”. Ici, chaque colonne prend un tiers.
  • gap: 24px ajoute 24 px entre les cartes, horizontalement et verticalement.

Responsive avec Grid

.services {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(260px, 1fr));
  gap: 24px;
}

C’est très puissant.

Cette ligne signifie :

  • crée autant de colonnes que possible,
  • chaque colonne fait minimum 260 px,
  • sinon elle prend une fraction de l’espace.

Explication détaillée de auto-fit et minmax()

  • auto-fit demande au navigateur de créer automatiquement le nombre de colonnes qui rentrent.
  • minmax(260px, 1fr) dit qu’une colonne doit faire au minimum 260 px.
  • Si plusieurs colonnes de 260 px rentrent, le navigateur les affiche côte à côte.
  • S’il n’y a pas assez de place, une carte passe à la ligne au lieu de déborder.
  • 1fr autorise les colonnes à s’élargir pour remplir l’espace restant.

Exemple concret

HTML :

<section class="services">
  <article class="card">
    <h3>Site vitrine</h3>
    <p>Idéal pour présenter ton activité.</p>
  </article>

  <article class="card">
    <h3>Landing page</h3>
    <p>Parfait pour vendre une offre précise.</p>
  </article>

  <article class="card">
    <h3>Maintenance</h3>
    <p>Pour garder ton site propre et sécurisé.</p>
  </article>
</section>

CSS :

.services {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(260px, 1fr));
  gap: 24px;
}

.card {
  padding: 24px;
  border-radius: 24px;
  background: white;
  box-shadow: 0 20px 60px rgba(15, 23, 42, 0.08);
}

Flexbox ou Grid ?

BesoinMeilleur choix
Aligner un menuFlexbox
Centrer un boutonFlexbox
Faire une grille de cartesGrid
Faire une mise en page complexeGrid
Aligner verticalementFlexbox
Créer un layout 2 colonnesGrid ou Flexbox

Mon avis : Pour un site moderne, utilise souvent Grid pour les grandes structures et Flexbox pour aligner le contenu à l’intérieur.

16. Responsive design

Un site responsive s’adapte aux écrans :

  • ordinateur,
  • tablette,
  • smartphone.

Règle de base

Toujours mettre dans le HTML :

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Exemple mobile first

On commence par le mobile, puis on améliore pour les grands écrans.

.hero {
  display: grid;
  gap: 32px;
  padding: 64px 24px;
}

.hero-title {
  font-size: 40px;
}

@media (min-width: 768px) {
  .hero {
    grid-template-columns: 1fr 1fr;
    align-items: center;
    padding: 96px 48px;
  }

  .hero-title {
    font-size: 64px;
  }
}

Comprendre chaque ligne

  • .hero cible la section principale de la page.
  • display: grid transforme la section en grille. Même avec une seule colonne, Grid prépare une structure propre.
  • gap: 32px ajoute 32 px d’espace entre les enfants de .hero, par exemple entre le texte et l’image.
  • padding: 64px 24px ajoute 64 px en haut et en bas, puis 24 px à gauche et à droite. C’est confortable sur mobile.
  • .hero-title cible uniquement le titre de la section hero.
  • font-size: 40px donne une taille de titre lisible sur téléphone.
  • @media (min-width: 768px) ouvre un breakpoint : les règles à l’intérieur s’appliquent seulement si l’écran fait au moins 768 px de large.
  • grid-template-columns: 1fr 1fr crée deux colonnes égales : une pour le texte, une pour l’image ou le visuel.
  • align-items: center aligne verticalement les deux colonnes au centre.
  • padding: 96px 48px augmente les espaces sur tablette et ordinateur, car il y a plus de place.
  • font-size: 64px agrandit le titre sur les grands écrans pour garder un impact visuel fort.

Pourquoi mobile first ?

Aujourd’hui, beaucoup de visiteurs arrivent par téléphone. Donc on crée d’abord une version mobile propre, puis on adapte pour ordinateur.

Breakpoints utiles

Un breakpoint n’est pas une règle magique liée à un appareil précis. C’est simplement un point où ton design commence à manquer de place ou, au contraire, peut utiliser plus d’espace. Les valeurs ci-dessous sont des repères courants, pas des obligations. 

/* Tablette */
@media (min-width: 768px) {
}

/* Desktop */
@media (min-width: 1024px) {
}

/* Grand écran */
@media (min-width: 1280px) {
}
BreakpointCe qu’on change souventExemple concret
min-width: 640pxPetites tablettesPasser une liste de cartes de 1 à 2 colonnes
min-width: 768pxTablettesMettre un hero en 2 colonnes
min-width: 1024pxOrdinateursAfficher une sidebar, agrandir les marges
min-width: 1280pxGrands écransLimiter le contenu avec max-width pour éviter les lignes trop longues

min-width ou max-width ?

Avec min-width, tu écris d’abord le style mobile, puis tu ajoutes les améliorations pour les écrans plus larges. C’est le plus simple pour apprendre et maintenir un site moderne. 

/* Style mobile par défaut */
.menu {
  display: grid;
  gap: 12px;
}

/* Style tablette et desktop */
@media (min-width: 768px) {
  .menu {
    display: flex;
    justify-content: flex-end;
  }
}

Avec max-width, tu écris plutôt le style desktop d’abord, puis tu corriges pour les petits écrans. Cette méthode existe, mais elle devient vite plus difficile quand le site grandit.

Images responsives

img {
  max-width: 100%;
  height: auto;
}

Explication ligne par ligne

  • img cible toutes les images de la page.
  • max-width: 100% empêche une image de dépasser la largeur de son parent.
  • height: auto garde les proportions de l’image pour éviter qu’elle soit écrasée.

Texte responsive avec clamp

.hero-title {
  font-size: clamp(2.5rem, 6vw, 5rem);
}

Cela veut dire :

  • minimum : 2.5rem,
  • idéal : 6vw,
  • maximum : 5rem.

Très utile pour les titres modernes.

Lecture complète : le titre ne descend jamais sous 2.5rem, essaie de suivre 6vw selon la largeur de l’écran, puis s’arrête à 5rem pour ne pas devenir trop grand.

Exemple concret : passer de 1 à 2 colonnes

Sur mobile, on empile le texte et l’image. Sur tablette et ordinateur, on les met côte à côte.

.about {
  display: grid;
  gap: 24px;
}

@media (min-width: 768px) {
  .about {
    grid-template-columns: 1fr 1fr;
    align-items: center;
  }
}

Explication ligne par ligne

  • .about cible la section “à propos”.
  • display: grid permet d’organiser ses éléments en grille.
  • gap: 24px garde un espace propre entre le texte et l’image.
  • Sans breakpoint, la grille reste naturellement sur une colonne si aucune colonne n’est déclarée.
  • @media (min-width: 768px) applique le changement à partir de la tablette.
  • grid-template-columns: 1fr 1fr place le contenu en deux colonnes égales.
  • align-items: center centre verticalement les éléments dans la section.

Erreurs fréquentes à éviter

  • Mettre width: 1200px sur un bloc principal.
  • Oublier max-width: 100% sur les images.
  • Faire des media queries avant d’avoir une version mobile correcte.
  • Utiliser une taille de texte trop petite sur téléphone.

17. Variables CSS

Les variables CSS permettent de garder une charte graphique propre.

Exemple

:root {
  --primary: #7c3aed;
  --secondary: #2563eb;
  --bg: #f8fafc;
  --surface: #ffffff;
  --text: #111827;
  --muted: #6b7280;
  --radius: 24px;
  --shadow: 0 24px 80px rgba(15, 23, 42, 0.12);
}

Utilisation

.card {
  background: var(--surface);
  color: var(--text);
  border-radius: var(--radius);
  box-shadow: var(--shadow);
}

Avantage concret

Si tu veux changer tout le violet du site, tu modifies seulement :

--primary: #7c3aed;

au lieu de chercher la couleur partout.

Démo variables CSS

Cette carte utilise var(--pro-brand).

18. Effets modernes : transitions, animations, ombres, gradients

Transition

.btn {
  transition: transform 0.2s ease, box-shadow 0.2s ease;
}

.btn:hover {
  transform: translateY(-2px);
  box-shadow: 0 16px 40px rgba(124, 58, 237, 0.25);
}

Gradient

.hero {
  background:
    radial-gradient(circle at top left, rgba(124, 58, 237, 0.25), transparent 35%),
    linear-gradient(135deg, #f8fafc, #eef2ff);
}

Texte en dégradé

.gradient-text {
  background: linear-gradient(135deg, #7c3aed, #2563eb);
  -webkit-background-clip: text;
  color: transparent;
}

HTML :

<h1>Votre site web <span class="gradient-text">professionnel</span></h1>

Animation simple

@keyframes float {
  0%, 100% {
    transform: translateY(0);
  }

  50% {
    transform: translateY(-12px);
  }
}

.mockup {
  animation: float 6s ease-in-out infinite;
}

Accessibilité : réduire les animations

Certaines personnes sont sensibles aux animations.

@media (prefers-reduced-motion: reduce) {
  * {
    animation: none !important;
    transition: none !important;
    scroll-behavior: auto !important;
  }
}

19. Positionnement CSS

Position static

Valeur normale par défaut.

.box {
  position: static;
}

Position relative

Permet de déplacer légèrement un élément.

.badge {
  position: relative;
  top: -4px;
}

Position absolute

Place un élément par rapport au parent positionné.

.card {
  position: relative;
}

.card::before {
  content: "";
  position: absolute;
  inset: 0;
  border-radius: 24px;
  background: linear-gradient(135deg, #7c3aed, #2563eb);
  opacity: 0.08;
}

Position fixed

Reste fixé à l’écran.

.whatsapp-button {
  position: fixed;
  right: 24px;
  bottom: 24px;
}

Position sticky

Reste collé après un certain scroll.

.site-header {
  position: sticky;
  top: 0;
  z-index: 50;
}

Z-index

Contrôle la profondeur.

.menu {
  position: fixed;
  z-index: 100;
}

Attention : z-index marche surtout sur les éléments positionnés.

21. Problématiques CSS concrètes

Problème 1 : mes cartes ne sont pas alignées

Cause fréquente

Tu utilises des marges approximatives.

Mauvais exemple

.card {
  display: inline-block;
  width: 30%;
  margin-right: 20px;
}

Correction avec Grid

.cards {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(260px, 1fr));
  gap: 24px;
}

Problème 2 : mon site déborde sur mobile

Cause fréquente

Un élément a une largeur fixe trop grande.

.hero-img {
  width: 900px;
}

Correction

.hero-img {
  max-width: 100%;
  height: auto;
}

Problème 3 : mon menu ne reste pas en haut

Correction

.site-header {
  position: sticky;
  top: 0;
  z-index: 100;
  backdrop-filter: blur(16px);
  background: rgba(255, 255, 255, 0.8);
}

Problème 4 : mes boutons ne font pas premium

Version basique

.btn {
  background: blue;
  color: white;
}

Version plus professionnelle

.btn {
  display: inline-flex;
  align-items: center;
  justify-content: center;
  gap: 8px;
  padding: 14px 22px;
  border-radius: 999px;
  border: 0;
  background: linear-gradient(135deg, #7c3aed, #2563eb);
  color: white;
  font-weight: 700;
  text-decoration: none;
  box-shadow: 0 16px 40px rgba(37, 99, 235, 0.25);
  transition: transform 0.2s ease, box-shadow 0.2s ease;
}

.btn:hover {
  transform: translateY(-2px);
  box-shadow: 0 24px 60px rgba(37, 99, 235, 0.32);
}

Problème 5 : mon texte est trop large et difficile à lire

Mauvais exemple

p {
  width: 100%;
}

Correction

p {
  max-width: 65ch;
}

ch correspond approximativement à la largeur d’un caractère. Un paragraphe entre 55 et 75 caractères de largeur est souvent plus lisible.

Problème 6 : mon style ne s’applique pas

Checklist de debug

  • Vérifie que le fichier CSS est bien lié avec <link>.
  • Vérifie que le nom de classe est identique dans le HTML et le CSS.
  • Regarde si une règle plus spécifique écrase ton style.
  • Inspecte l’élément dans le navigateur pour voir quelle règle gagne.

Exemple courant

<button class="button-primary">Envoyer</button>
.btn-primary {
  background: #2563eb;
}

Ici, le CSS ne marche pas car le HTML utilise button-primary alors que le CSS cible btn-primary.

Problème 7 : mes espacements sont incohérents

Choisis une échelle simple et réutilise-la partout : 4, 8, 12, 16, 24, 32, 48, 64 px.

.section { padding-block: 64px; }
.card { padding: 24px; }
.stack { display: grid; gap: 16px; }
Le CSS devient plus propre quand les décisions se répètent volontairement au lieu d’être improvisées élément par élément.
