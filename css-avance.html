1. CSS Grid avancé

Grid permet des mises en page complexes en 2 dimensions : lignes, colonnes, zones nommées et placement explicite. 

.layout {
  display: grid;
  grid-template-columns: 200px 1fr 1fr;
  grid-template-rows: auto 1fr auto;
  grid-template-areas:
    "nav  header header"
    "nav  main   side"
    "nav  footer footer";
  gap: 12px;
}
.header { grid-area: header; }
.nav    { grid-area: nav; }
.main   { grid-area: main; }
.side   { grid-area: side; }
.footer { grid-area: footer; }

2. Flexbox avancé

Au-delà de justify-content, Flexbox gère le wrapping, le ratio de croissance (flex) et l’alignement par item. 

.toolbar {
  display: flex;
  flex-wrap: wrap;
  gap: 8px;
  align-items: center;
}
.toolbar .grow { flex: 1 1 180px; }
.toolbar .fixed { flex: 0 0 auto; }

3. Variables CSS

Les custom properties (--nom) permettent des thèmes dynamiques et une cohérence de design system. 

:root {
  --brand: #0f766e;
  --radius: 12px;
}
[data-theme="dark"] {
  --brand: #2dd4bf;
  --surface: #0f172a;
}
.btn {
  background: var(--brand);
  border-radius: var(--radius);
}

4. Container Queries

Contrairement aux media queries (viewport), les container queries adaptent le style selon la taille du conteneur parent. 

.card-wrap {
  container-type: inline-size;
}
@container (min-width: 360px) {
  .card-inner {
    display: grid;
    grid-template-columns: 80px 1fr;
  }
}

5. Cascade Layers (@layer)

Les layers contrôlent la priorité des styles indépendamment de l’ordre et de la spécificité classique. 

@layer reset, base, components, utilities;

@layer base {
  button { background: gray; }
}
@layer components {
  .btn { background: blue; } /* gagne sur base */
}
@layer utilities {
  .bg-red { background: red; } /* priorité max */
}
  • L’ordre de déclaration des layers définit la priorité
  • Idéal pour séparer reset / composants / utilitaires

6. Animations complexes

Keyframes, transitions et timing-functions créent des micro-interactions fluides. Respecte toujours prefers-reduced-motion. 

@keyframes float {
  0%, 100% { transform: translateY(0); }
  50%      { transform: translateY(-8px); }
}
.orb {
  animation: float 2s ease-in-out infinite;
  transition: box-shadow 0.25s ease;
}
@media (prefers-reduced-motion: reduce) {
  .orb { animation: none; }
}

7. CSS Houdini

Houdini expose des APIs bas niveau pour étendre le moteur CSS (Paint Worklet, Properties & Values, Layout…). 

// Enregistrer une propriété typée
CSS.registerProperty({
  name: "--angle",
  syntax: "<angle>",
  inherits: false,
  initialValue: "0deg"
});

// Paint Worklet (support variable selon navigateurs)
CSS.paintWorklet.addModule("checkerboard.js");
.box { background: paint(checkerboard); }
Support encore partiel selon les navigateurs : vérifie la compatibilité avant un usage en production.

8. Fonctions modernes

clamp(), min(), max() et calc() rendent le design fluide sans media queries partout. 

h1 {
  font-size: clamp(1.5rem, 4vw + 0.5rem, 3rem);
}
.sidebar {
  width: min(100%, 320px);
}
.card {
  padding: max(1rem, 2vw);
  width: calc(100% - 2rem);
}

Démo — clamp() sur la taille du texte

Ce texte utilise clamp(1rem, 2.5vw + 0.5rem, 1.75rem). Redimensionne la fenêtre pour voir l’effet.

9. Subgrid

subgrid aligne les grilles imbriquées sur les pistes de la grille parente — parfait pour des cartes de hauteurs égales. 

.parent {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 16px;
}
.card {
  display: grid;
  grid-template-rows: subgrid;
  grid-row: span 3; /* titre + texte + bouton */
}

10. Sélecteurs avancés

:has(), :is(), :where() et :not() simplifient des règles autrefois impossibles en CSS pur. 

/* Parent qui contient un input invalide */
.field:has(input:invalid) { border-color: red; }

/* Grouper sans répéter */
:is(h1, h2, h3) { line-height: 1.2; }

/* Même chose, spécificité 0 */
:where(.card, .panel) { margin-block: 1rem; }

/* Exclure */
button:not(:disabled) { cursor: pointer; }
