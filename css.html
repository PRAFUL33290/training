Les bases de CSS

CSS donne le style à ta page : couleurs, tailles, espacements et mise en page.

1. Qu’est-ce que CSS ?

Le CSS s’applique aux éléments HTML. Une règle a un sélecteur, puis des propriétés.

En pratique, le HTML décrit le contenu et le CSS décide comment ce contenu doit apparaître à l’écran. Par exemple, le HTML dit “ceci est un titre”, et le CSS dit “ce titre est grand, bleu et espacé du paragraphe suivant”. 

/* sélecteur { propriété: valeur; } */
p {
    color: blue;
    font-size: 18px;
}

Explication ligne par ligne

  • p sélectionne tous les paragraphes de la page.
  • { ouvre le bloc de styles appliqué aux paragraphes.
  • color: blue; met la couleur du texte en bleu.
  • font-size: 18px; règle la taille du texte à 18 pixels.
  • } ferme le bloc de styles.
Lecture de la règle : “pour tous les paragraphes p, mets le texte en bleu et donne-lui une taille de 18 pixels”.

2. Lier le CSS au HTML

La méthode recommandée : un fichier externe.

<!-- Dans le <head> du HTML -->
<link rel="stylesheet" href="styles.css">

Cette ligne indique au navigateur : “charge aussi le fichier styles.css pour savoir comment afficher la page”. Si le fichier est dans un dossier, il faut écrire le bon chemin : 

<link rel="stylesheet" href="css/styles.css">
<link rel="stylesheet" href="assets/css/main.css">

Ou en CSS interne :

<style>
    h1 { color: red; }
</style>
Pour un site complet, garde le CSS dans un fichier externe : c’est plus lisible, plus facile à corriger et réutilisable sur plusieurs pages.

3. Sélecteurs essentiels

Un sélecteur sert à choisir précisément ce que tu veux styliser. Le plus important est de savoir quand utiliser une balise, une classe ou un ID. 

/* Balise */
h1 { color: black; }

/* Classe (réutilisable) */
.titre { font-size: 24px; }

/* ID (unique) */
#logo { width: 100px; }

/* Combinaison */
.card p { color: #666; }

Explication ligne par ligne

  • /* Balise */ est un commentaire : il sert à expliquer le code, le navigateur l’ignore.
  • h1 { color: black; } cible tous les titres h1 et met leur texte en noir.
  • /* Classe (réutilisable) */ annonce l’exemple avec une classe.
  • .titre { font-size: 24px; } cible les éléments avec class="titre" et met leur texte à 24 px.
  • /* ID (unique) */ annonce l’exemple avec un identifiant unique.
  • #logo { width: 100px; } cible l’élément avec id="logo" et lui donne une largeur de 100 px.
  • /* Combinaison */ annonce un sélecteur qui combine plusieurs informations.
  • .card p { color: #666; } cible seulement les paragraphes qui se trouvent dans un élément .card.
Sélecteur Quand l’utiliser Exemple concret
h1 Pour tous les titres principaux Uniformiser la taille des titres de page
.btn Pour un style réutilisable Styliser plusieurs boutons du site
#contact Pour une zone unique Créer un lien vers la section contact
.card p Pour cibler un élément dans un bloc Changer seulement les paragraphes des cartes

4. Couleurs et textes

La lisibilité vient surtout de trois choix : une couleur de texte assez contrastée, une taille confortable et une hauteur de ligne suffisante. 

body {
    color: #1a1a1a;          /* texte */
    background-color: #fff;  /* fond */
    font-family: Arial, sans-serif;
    font-size: 16px;
    line-height: 1.6;
}

.titre {
    font-weight: bold;
    text-align: center;
}

Exemple concret : carte de présentation

.profil-card {
    background: #ffffff;
    color: #0f172a;
    font-family: Arial, sans-serif;
    line-height: 1.6;
}

.profil-card .role {
    color: #64748b;
    font-size: 14px;
}

Explication ligne par ligne

  • .profil-card cible la carte de présentation.
  • background: #ffffff donne un fond blanc à la carte.
  • color: #0f172a définit une couleur de texte sombre et lisible.
  • font-family choisit la police utilisée dans la carte.
  • line-height: 1.6 augmente l’espace entre les lignes pour améliorer la lecture.
  • .profil-card .role cible seulement l’élément role placé dans la carte.
  • color: #64748b rend le rôle plus discret que le nom.
  • font-size: 14px réduit légèrement la taille du rôle.

5. Box model (le modèle de boîte)

Chaque élément est une boîte avec contenu, padding, bordure et marge.

  • content : le texte ou l’image à l’intérieur.
  • padding : l’espace entre le contenu et la bordure.
  • border : le contour de la boîte.
  • margin : l’espace entre cette boîte et les autres éléments.
.boite {
    width: 300px;
    padding: 20px;        /* intérieur */
    border: 2px solid #2563eb;
    margin: 16px;         /* extérieur */
    border-radius: 8px;
    box-sizing: border-box;
}

Explication ligne par ligne

  • .boite cible tous les éléments qui ont la classe boite.
  • width: 300px donne une largeur de base de 300 pixels.
  • padding: 20px ajoute de l’espace à l’intérieur de la boîte.
  • border: 2px solid #2563eb ajoute une bordure bleue de 2 pixels.
  • margin: 16px ajoute de l’espace autour de la boîte.
  • border-radius: 8px arrondit légèrement les coins.
  • box-sizing: border-box inclut le padding et la bordure dans la largeur totale.

Exemple de boîte

Je suis une boîte CSS
Conseil pratique : ajoute souvent box-sizing: border-box. Ainsi, une boîte de 300 px reste à 300 px même avec du padding et une bordure.

6. Flexbox (mise en page)

Flexbox aligne facilement des éléments en ligne ou en colonne.

.row {
    display: flex;
    justify-content: space-between; /* horizontal */
    align-items: center;            /* vertical */
    gap: 16px;
}

Explication ligne par ligne

  • .row cible un conteneur qui doit aligner plusieurs éléments.
  • display: flex place les enfants sur un axe flexible.
  • justify-content: space-between pousse les éléments aux extrémités avec de l’espace entre eux.
  • align-items: center centre les éléments verticalement.
  • gap: 16px garde toujours 16 px entre les éléments.

.nav {
    display: flex;

8. Breakpoints et responsive

Un breakpoint est une largeur d’écran à partir de laquelle tu changes la mise en page. Par exemple, un site peut afficher une seule colonne sur téléphone, puis deux colonnes à partir de 768 px.

En CSS, on écrit un breakpoint avec @media. Le code placé dans le bloc ne s’applique que si la condition est vraie. 

.cards {
    display: grid;
    gap: 16px;
}

@media (min-width: 768px) {
    .cards {
        grid-template-columns: repeat(2, 1fr);
    }
}

Explication ligne par ligne

  • .cards { ouvre une règle pour le bloc qui contient les cartes.
  • display: grid; active CSS Grid pour organiser les cartes.
  • gap: 16px; ajoute un espace régulier entre les cartes.
  • } ferme le style mobile de base.
  • @media (min-width: 768px) { ouvre un breakpoint qui s’active à partir de 768 px.
  • .cards { rouvre une règle pour les cartes, mais seulement dans ce breakpoint.
  • grid-template-columns: repeat(2, 1fr); crée deux colonnes de largeur égale.
  • } ferme la règle .cards du breakpoint.
  • } ferme le bloc @media.
Méthode conseillée : commence par le mobile sans media query, puis ajoute des breakpoints pour tablette et ordinateur. C’est ce qu’on appelle le mobile first.

9. Comprendre display avec une démo

La propriété display décide comment un élément se comporte dans la page : est-ce qu’il prend toute la ligne, est-ce qu’il reste dans le texte, est-ce qu’il organise ses enfants en ligne ou en grille ? 

.demo-block {
    display: block;
}

.demo-inline {
    display: inline;
}

.demo-inline-block {
    display: inline-block;
}

.demo-flex {
    display: flex;
}

.demo-grid {
    display: grid;
}

.demo-flexbox {
    display: flexbox; /* incorrect */
}

Explication ligne par ligne

  • .demo-block { ouvre une règle pour l’exemple block.
  • display: block; force l’élément à prendre toute la largeur disponible.
  • } ferme la règle .demo-block.
  • .demo-inline { ouvre une règle pour l’exemple inline.
  • display: inline; garde l’élément dans le flux du texte, sans largeur ni hauteur contrôlables normalement.
  • } ferme la règle .demo-inline.
  • .demo-inline-block { ouvre une règle pour l’exemple inline-block.
  • display: inline-block; garde l’élément sur la ligne, mais permet d’utiliser padding, width et height.
  • } ferme la règle .demo-inline-block.
  • .demo-flex { ouvre une règle pour l’exemple Flexbox.
  • display: flex; active Flexbox pour aligner les enfants sur un axe.
  • } ferme la règle .demo-flex.
  • .demo-grid { ouvre une règle pour l’exemple Grid.
  • display: grid; active CSS Grid pour organiser les enfants en lignes et colonnes.
  • } ferme la règle .demo-grid.
  • .demo-flexbox { ouvre une règle qui montre une erreur fréquente.
  • display: flexbox; est incorrect : cette valeur n’existe pas en CSS moderne.
  • } ferme la règle .demo-flexbox.
Attention : on dit souvent “Flexbox” pour parler du système de mise en page, mais la ligne CSS correcte est display: flex, pas display: flexbox.
