1. Web Components

Les Web Components regroupent Custom Elements, Shadow DOM et Templates pour créer des composants HTML réutilisables, sans framework.

  • customElements.define() — enregistre une balise
  • attachShadow() — isole le style et le DOM
  • <template> — modèle HTML clonable
  • <slot> — injection de contenu externe
<!-- Utilisation d'un composant -->
<user-card name="Alice" role="Dev"></user-card>

2. Shadow DOM

Le Shadow DOM isole le HTML et le CSS d’un composant pour éviter les fuites de styles vers (et depuis) le reste de la page. 

class InfoBox extends HTMLElement {
  constructor() {
    super();
    const root = this.attachShadow({ mode: "open" });
    root.innerHTML = `
      <style>
        .box {
          padding: 12px 16px;
          border-radius: 10px;
          background: #eef2ff;
          color: #3730a3;
          font: 500 0.95rem system-ui, sans-serif;
        }
      </style>
      <div class="box">Style encapsulé (Shadow DOM)</div>
    `;
  }
}
customElements.define("info-box", InfoBox);

3. Custom Elements

Les Custom Elements permettent de créer ses propres balises HTML (ex. <my-card>). Le nom doit contenir un tiret. 

class MyCard extends HTMLElement {
  connectedCallback() {
    const title = this.getAttribute("title") || "Carte";
    this.innerHTML = `
      <article class="card-like">
        <strong>${title}</strong>
        <p>Composant autonome</p>
      </article>
    `;
  }
}
customElements.define("my-card", MyCard);

4. Templates (<template>)

Un <template> contient du HTML non rendu jusqu’à ce qu’on le clone dynamiquement avec JavaScript. 

<template id="item-tpl">
  <li class="tpl-item"><span></span></li>
</template>

<script>
  const tpl = document.getElementById("item-tpl");
  const clone = tpl.content.cloneNode(true);
  clone.querySelector("span").textContent = "Nouvel item";
  list.appendChild(clone);
</script>

    5. Slots

    Les <slot> injectent du contenu fourni par l’utilisateur dans le Shadow DOM d’un Web Component. 

    <!-- Dans le Shadow DOM -->
<h3><slot name="title">Titre par défaut</slot></h3>
<div><slot>Contenu par défaut</slot></div>

<!-- Utilisation -->
<slot-card>
  <span slot="title">Mon titre</span>
  <p>Contenu projeté dans le slot</p>
</slot-card>

    6. ARIA

    Les attributs ARIA (Accessible Rich Internet Applications) améliorent l’accessibilité quand le HTML sémantique ne suffit pas.

    • role — rôle de l’élément pour les lecteurs d’écran
    • aria-label — nom accessible
    • aria-expanded — état ouvert / fermé
    • aria-live — annonce des mises à jour dynamiques
    <button
  aria-expanded="false"
  aria-controls="panel"
  id="toggle">
  Afficher les détails
</button>
<div id="panel" hidden>Contenu caché…</div>

    Démo — panneau accessible

    Ce panneau est annoncé correctement grâce à aria-expanded et aria-controls.

    8. Lazy Loading

    Le chargement différé des images et iframes améliore les performances initiales de la page. 

    <img
  src="photo.jpg"
  alt="Paysage"
  loading="lazy"
  width="800"
  height="450"
>

<iframe
  src="https://exemple.com"
  loading="lazy"
  title="Carte"
></iframe>
    • loading="lazy" — charge hors écran plus tard
    • Indique toujours width / height pour éviter le layout shift
    • Utile pour les galeries et le bas de page

    9. Progressive Web Apps
