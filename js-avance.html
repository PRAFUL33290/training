1. Closures

Une closure est une fonction qui conserve l’accès à son contexte lexical (variables externes) même après que la fonction parente a terminé. 

function createCounter() {
  let count = 0; // privé
  return function () {
    count += 1;
    return count;
  };
}
const next = createCounter();
next(); // 1
next(); // 2

2. Prototype et héritage

Chaque objet JS a un lien interne vers un prototype. Les propriétés manquantes sont cherchées dans la chaîne de prototypes. 

function Animal(name) {
  this.name = name;
}
Animal.prototype.speak = function () {
  return this.name + " fait un bruit";
};

class Dog extends Animal {
  speak() {
    return this.name + " aboie";
  }
}
const d = new Dog("Rex");
d.speak(); // "Rex aboie"
  • Object.getPrototypeOf(obj) — prototype d’un objet
  • Les class ES6 sont du sucre syntaxique sur les prototypes

3. Event Loop

L’event loop gère l’asynchrone : call stack, microtasks (Promises) et macrotasks (setTimeout, I/O…). 

console.log("A");
setTimeout(() => console.log("B"), 0);      // macrotask
Promise.resolve().then(() => console.log("C")); // microtask
console.log("D");
// Ordre : A, D, C, B

4. Promises

                    

                        Une Promise représente une valeur future. Elle est
                        pending, puis fulfilled ou rejected.
                    

                    
function wait(ms) {
  return new Promise((resolve) => setTimeout(resolve, ms));
}

wait(500)
  .then(() => "OK")
  .then((msg) => console.log(msg))
  .catch((err) => console.error(err));

5. Async / Await

                    

                        async/await est une syntaxe moderne basée sur les Promises,
                        plus lisible que les chaînes de .then().
                    

                    
async function loadUser(id) {
  try {
    const res = await fetch("/api/users/" + id);
    if (!res.ok) throw new Error("Erreur réseau");
    return await res.json();
  } catch (err) {
    console.error(err);
  }
}

7. Proxy

                    

                        Un Proxy intercepte les opérations sur un objet
                        (lecture, écriture, keys…) via des traps.
                    

                    
const user = { name: "Alice" };
const proxied = new Proxy(user, {
  get(target, prop) {
    console.log("lecture", prop);
    return target[prop];
  },
  set(target, prop, value) {
    if (prop === "age" && value < 0) throw new Error("Âge invalide");
    target[prop] = value;
    return true;
  }
});
proxied.name; // log "lecture name"

8. Generators

                    

                        Les generators (function*) peuvent être mis en pause
                        avec yield puis repris.
                    

                    
function* ids() {
  let i = 1;
  while (true) {
    yield i++;
  }
}
const gen = ids();
gen.next().value; // 1
gen.next().value; // 2
